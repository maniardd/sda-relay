Off-boarding Process for Catalyst Cloud Management Devices in Monitor-Only Mode in Meraki Dashhboard

  1. Last updated
  2. Save as PDF
  1. Overview
  2. Offboarding Workflow

 

Overview

The offboarding process for Cisco Catalyst deployments managed through Meraki Dashboard in monitoy-only mode has been redesigned to provide a streamlined, regional-aware, and automated clean-up experience. 

This update applies to Meraki Dashboard monitor-only deployments across the primary dashboard infrastructure and regional cloud environments, including India, Canada, and China. 

  • Catalyst switches (operating in device configuration mode)

  • Wireless LAN Controllers (WLCs) - (Includes support for India and Canada regions)

  • Routers 

Offboarding Workflow

When a device is removed from a Meraki Dashboard network, the following automated process occurs:

  1. Removing a device from the dashboard network immediately triggers an automated offboarding job.

    • The cleanup process begins without requiring manual intervention.

    • Configuration artifacts related to cloud onboarding and telemetry are targeted for removal.

  1. Devices undergoing removal display a “Removing” tag next to the device name in the dashboard.

    • This provides visibility into the removal progress, so you always know the state of your device. 

  1. Cleanup scripts are delivered directly from the device’s assigned Dashboard cloud cluster (India, Canada, China, or the primary Dashboard infrastructure).

    • No cleanup traffic is routed through remote or cross-region clusters.

  1. After cleanup is completed, the device is removed from the dashboard network. 

When a Catalyst device using cloud management with monitor-only or device configuration is removed from the dashboard, onboarding and telemetry configurations will be removed from the device by configuring and executing an EEM Script. 

Line VTY configurations will not be removed automatically. After offboarding completes, administrators must manually remove these lines.

Use the following command format:
no line vty [x] [y]

The exact values for the line numbers will vary but will contain four total lines inclusive in the range. 

Refer to the full details on the additions and updates applied to a switch enabled for cloud management with device configuration: Cloud Management with Device Configuration – Required Modifications.
