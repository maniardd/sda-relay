Upgrading Cloud-Monitored Switches to Cloud Management with Device Configuration

  1. Last updated
  2. Save as PDF

  2. Step 2: Remove switches from Dashboard network and enable cloud management with device configuration
  3. Step 3: Select devices to add to network as cloud managed switches with device configuration
  4. Alternate option: Manual Process

Cloud Monitoring end of service extended to March 31, 2026: The end-of-service date for Cloud Monitoring has been extended by two months from January 31, 2026 to March 31, 2026 to provide additional time to complete the upgrade process. All other terms remain unchanged. Learn more.

Help us improve your experience. Complete this quick survey to share your feedback on Cloud Monitoring or cloud management with device configuration.

If you have switches currently on Cloud Monitoring that need to be upgraded to cloud management with device configuration, you will see this alert at the top of your Switch list page:

 

Clicking Get Started in the Upgrade devices section will display the upgrade page for cloud-monitored switches.

Before proceeding to the next section, review the following:

  • Review the IOS XE image upgrade documentation for Cloud Monitoring on Catalyst switches, including prerequisites and expected outcomes. 
  • Ensure sufficient free flash memory is available on the switch before initiating the upgrade to prevent failures. For more information, refer to the 'Preparing for IOS XE upgrades' documentation.
  • Review the Known Issues section for device configuration mode prior to deployment.
  • All steps to complete the upgrade to cloud management with device configuration should be completed at the same time. Upgrading only IOS XE without completing all steps below will result in limited information displayed in Dashboard. IOS XE 17.15.3 and above is only supported using cloud management with device configuration.
  • The 17.15.4b image is now available on the Meraki dashboard for migrating cloud monitored switches to cloud management with device configuration. This image includes the fix for the known SNMP vulnerability CSCwq31287.
    • Note: Upgrades to IOS XE 17.15.4b from versions 17.15.3 or 17.15.4 are not supported through the Meraki dashboard.

 

Step 1: Upgrade switches to IOS XE 17.15.3, 17.15.4, or 17.15.4b using Dashboard

Within Dashboard, you can use the firmware upgrade option to upgrade your switch(es) to IOS XE 17.15.3, 17.15.4, or 17.15.4b. 

You can select the desired version and then schedule the upgrade.

The switch will upgrade and update when complete:

 

 

Step 2: Remove switches from Dashboard network and enable cloud management with device configuration

Navigate to your switch list by clicking Switching > Switches from the left-side navigation.

Switches need to be removed from Dashboard to disable Cloud Monitoring. You will be prompted to select that you would like to enable the tunnel on the switch as part of this process to automate enabling cloud management with device configuration.

To begin the process, click Get Started in the Transition Devices section on the right side.

By default, all eligible switches will be selected for removal when clicking Get Started.

If you would like to begin the transition process for only selected switches, check the box next to each switch you would like to remove and then click Remove at the top right corner of the switch list. You can then continue below.

Make sure to keep the box checked to allow Dashboard to configure Service Meraki Connect to complete the upgrade process.

Unchecking this box and removing the device will disable the switch tunnel and disconnect the switch from Dashboard.

 

Before selecting Remove, you can click Download device list to save a CSV file locally with the list of switches eligible for reference later.

 

Step 3: Select devices to add to network as cloud managed switches with device configuration
  • When adding stacked switches, all members of the stack must be selected to add to network.
  • Organizations using per-device licensing must use the network-wide add devices page.

After removing the switch from Cloud Monitoring, go to Organization > Inventory from the left-side navigation.

Select which switches you would like to enable for cloud management and click Add to Network. If needed, you can use the saved device list to search by MAC address or name.

 

 

Complete the three steps to add the switch to your Dashboard network using device configuration:

 

 

You can then go back to your switch list page by selecting Switching > Switches from the left-side navigation and confirm the switch was added with device configuration.

 

Alternate option: Manual Process

Click 日本語 for Japanese

If you decide not to use the above process, you can remove and upgrade your switches before adding them back to Dashboard.

  1. Remove the desired switch(es) from Dashboard networks. For additional detail, see Offboarding/Removing Switches from Cloud Monitoring.

  2. Wait 10-15 minutes for device-side configuration cleanup to complete.

  3. Verify pre-requisites for device configuration.

  4. Upgrade to IOS XE 17.15.3, 17.15.4, 17.15.4b or 17.15.5 if running an earlier version.

  5. Enable cloud management with device configuration.
