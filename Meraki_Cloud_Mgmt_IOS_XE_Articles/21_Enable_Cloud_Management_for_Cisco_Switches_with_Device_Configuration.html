This guide provides an overview and a step-by-step guide on how to connect Cisco Catalyst 9000 switches and Cisco C9000 Smart Switches to cloud management with configuration source: device.

Help us improve your experience. Complete this quick survey to share your feedback on Cloud Monitoring or Configuration Source: Device.

Before starting, verify that all prerequisites are fulfilled, such as supported hardware, the appropriate IOS XE version, and the necessary firewall rules. For more information, refer to Catalyst Cloud-Managed Switches Overview.

Organizations using per-device licensing must use the network-wide add devices page to onboard their Catalyst switches.

Please note that for Cisco C9000 Smart Switches, the command lines are different from those of Catalyst 9000 switches: 

  • show cloud-mgmt: to retrieve Cloud ID or confirm cloud tunnel connection status
  • service cloud-mgmt connect: to initiate communication with cloud 
Prerequisites for Onboarding a Cisco Catalyst or Smart Switch in Device Configuration Mode:
  • Uplink connectivity (from the switch to the cloud and not connectivity to the computer being used) must be via a front-panel port (not the management interface).
  • Only the default VRF is supported.
  • Ensure routes are in place to reach external addresses including a default route (use of ip default-gateway is not supported).
  • IP routing (ip routing) must be enabled on the switch.
  • AAA on the switch must be configured using aaa new-model.
  • AAA authentication must permit local login and authorization must allow exec for local accounts. This permits the automatically provisioned service account to access the switch from Dashboard following initial onboarding.
  • The user account for onboarding must have privilege-15 level access on the switch.

If TACACS or RADIUS authorization is used, local must be first in the authorization list prior to TACACS or RADIUS servers.

Verify Meraki Tunnel Prerequisites on the switch.

  • The Meraki Tunnel only supports the Global VRF.

  • Domain Name Lookup is required for hostname resolution to the Dashboard Registration and Meraki Tunnel services.

  • ip name-server {Domain server IP address}
  • ip domain lookup

  • The switch clock must reflect the correct current time in order to establish a mutual TLS tunnel with the Registration and Meraki Tunnel services. Enabling NTP is recommended to accomplish this.

  • ntp server {ntp server IP address}

Prior to adding a switch in device configuration mode, ensure it is disconnected from its current cloud connection, such as Cloud Monitoring. Switches that are currently connected to the Dashboard via Cloud Monitoring have to be removed from their Dashboard network before they can be enabled for cloud management with device configuration. For detailed instructions, refer to Upgrading Cloud-Monitored Switches to Cloud Management with Device Configuration.

The Meraki tunnel for modular switches will form only when the chassis and supervisor(s) have been added to a dashboard network.

Known Issues and Limitations

Running show ip http server all May Cause SSH Session Disruption (IOS XE 17.15.3, 17.15.4, and 17.15.4b): 

  • Running the show ip http server all command on Cisco Catalyst 9200, 9300, and 9500 Series switches running IOS XE 17.15.3, 17.15.4, and 17.15.4b versions may cause all SSH sessions to hang or drop and prevent new SSH connections. All vty lines may become occupied and cannot be cleared without rebooting the switch. This can result in reporting the switch as offline. Avoid running this command in production environments. 
  • This issue is resolved in IOS XE 17.15.5 and will be included in the future IOS XE 17.18.3 and subsequent versions.

Onboarding Limit for Catalyst Devices (Configuration Source: Device)

When onboarding Catalyst devices in configuration source: device, onboarding more than 10 switches may fail with an 'Internal Server Error'. To resolve this, onboard switches in batches of 10 or fewer.

Step 1: Initiate Registration with the Cloud

To begin, the switch-side service in IOS XE must be enabled to initiate communication with the cloud.

While in configuration mode on the switch, enter either command: 

  • For Catalyst 9000 switches: service meraki connect
  • or For Cisco 9000 smart switches: service cloud-mgmt connect

Example:

C9300-switch#conf t
Enter configuration commands, one per line. End with CNTL/Z.
C9300-switch(config)#service meraki connect
C9300-switch(config)#end
  • After enabling the service for the first time, the tunnel may take few minutes to connect.
  • To confirm the Catalyst 9000 switch has successfully registered and the Meraki Tunnel is connected, use: 
    • For Catalyst 9000 switches: show meraki connect command; 
    • Or for Cisco 9000 smart switches: show cloud-mgmt
  • The tunnel state should be Up.
  • If the tunnel is not up, make sure to verify the correct firewall rules are in place. Additional details on connection attempts are available in the syslog (show logging).
  • The output will provide the required Cloud ID to claim the switch to a Dashboard network.

For example, the output of a successful connection may appear similar to:


 

The Cloud ID in the last section for Meraki Device Registration will be used to claim and register the switch in Dashboard. 

Please note, for Cisco 9350 smart switches, your switch comes with CloudID labels on the box and order confirmation email. The Cloud ID from your device command and the label will match,  so you can use either for the next step. 

For C9610, please follow step 1 above to retrieve Cloud ID from command before claiming to the inventory. 

  • When adding stacked switches, all members of the stack should be selected to add to network.
  • Organizations using per-device licensing must use the network-wide add devices page.

Step 2: Onboard the Switch to Meraki Dashboard Using Cloud ID

To continue, log in to Dashboard and navigate to Organization > Inventory.

  1. Click Claim Devices.


     
  2. Click Claim Individual Devices.



    Note: You can enter one or more Cloud IDs from your Catalyst switches. 

          
  3. After claiming the devices, on the Inventory page, select the checkbox next to the switches you would like to enable for cloud management.

Modular switch chassis and supervisors have separate unique cloud IDs. All modular switch cloud IDs must be added to the same network.

When adding Catalyst switch stacks to a network, make sure to include all stack members.

  1. In the Add devices to network window, choose the Dashboard Network for your switch and click Next.


 

  1. Select Device configuration and enter level-15 privilege credentials for the switch(es).

Note: Credentials are used to connect to your switch and update the configuration for use with device configuration.

The configuration lines will be automatically applied to your switch are available here: Cloud Management with Device Configuration Required Modification.

After confirming the details and completing the process, the configuration will be automatically applied to your switch.

You can view your switch(es) in the Dashboard by navigating to Switching > Switches. If you have added your switches, an alert will appear for a few minutes while the setup finalizes. Allow up to 15 minutes for the process to complete and for the data to update in the Dashboard.
